Functions

Contents

This describes the API functions exposed through the "FF" web applications. See the request URLs documentation for information about how to call these functions, and see the return objects documentation for information about how to handle the responses. As mentioned in the request URLs documentation, all function parameters are encoded as HTTP GET parameters unless otherwise stated.

Each function entry describes the name of the function (to be used as the ACTION parameter in the request), the purpose of the function, the output object type and any additional output value notes, the function parameters, and the permissions required to execute the function.

For example, the "login" function described below would be used like this:

http://<facefirst_host>/FF?ACTION=login&username=Bob&password=abc123

The following columns are in the Parameters table:

  • Parameter - The name of the function parameter.
  • Type - Parameter type and a link to the type documentation.
  • Count - Parameter count. This is the number of times the parameter may be specified. This will be either a single integer, or a range. Examples:
    • 1 - Exactly once, parameter is required.
    • 0..1 - Zero or one times, parameter is optional.
    • 0..* - Zero or more times.
    • 1..* - One or more times.
    • ? - A special case, details are specified in description.
  • Description - Description of the parameter.

Important: Some functions can return more than one return object type. In all such cases, a function either returns the requested data, or a SimpleResult with the "status" set to "error". To determine whether the returned data is either the requested data or a SimpleResult, check the root element of the returned data -- if it is "result" then it's a SimpleResult, otherwise it's something else (e.g. in JavaScript check if "theReturnedObject.result" is undefined).

login

Description
Log a user in. The back-end web server uses cookies to track session IDs.
Output
LoginSimpleResult

The "status" field will be "ok" if the user was succesfully logged in, "error" if an error occured, or "wantKey" if the user has the secure-login option enabled. If the status is "wantKey", a randomly generated key is emailed to the user's alert email address and the front-end must display an input that allows the user to enter the key. In that case, the key must be passed to the "loginKey" function to complete the login process. Note that secure FaceFirst authentication can not be used from an edge server. If the status is "ok", the "cookies" field will contain session and device id cookies; these name/value pairs should be submitted as cookies for all additional Web API requests using this session.

Parameters
Parameter Type Count Description
username string 1 User name.
password string 1 Password.
device_id string 0..1 The id of the device; if known at login can be passed in on the command line or as the DEVICE_ID cookie.
Permissions Required
  • The user (and the account that the user belongs to) must be enabled.
