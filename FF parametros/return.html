Return Objects

Contents

This describes the JSON schema for data returned from the back-end server. XML data is also available (see request URLs) but the XML schema is not documented here.

Object structure is documented like this:

  • car
    • color
      • trim ( color )
      • body ( color )
      • interior
        • seats ( color )
        • floor ( color )
    • maxHorsepower ( decimal )
    • repairs[] ( string )
    • previousOwners[]
      • firstName ( string )
      • lastName ( string )

Fields are indicated in bold, a [] indicates an array, the type of the field is indicated in parentheses. The above structure would describe the following fields:

  • car.color.trim
  • car.color.body
  • car.color.interior.seats
  • car.color.interior.floor
  • car.maxHorsepower
  • car.repairs[]
  • car.previousOwners[].firstName
  • car.previousOwners[].lastName

Following the structure is a description of each field:

  • trim - Color of trim.
  • body - Color of body.
  • seats - Color of seats.
  • floor - Color of floor.
  • maxHorsepower - Peak horsepower.
  • repairs - Array of strings describing all repairs done to the car.
  • firstName - A previous owner's first name.
  • lastName - A previous owner's last name.

By default, returned JSON data will be wrapped in parentheses, ready to be passed as an argument to a single-argument JavaScript function (other options for return format are available, see request URLs for more info). Example:

({"car": {
  "color": {
    "trim": "Sky Brown",
    "body": "Fire Engine Black",
    "interior": {
      "seats": "Hot Gray",
      "floor": "Imperial Zucchini"
    }
  },
  "maxHorsepower": 320.2,
  "repairs": [
    "Rebuilt turboencabulator",
    "Minor adjustment to utility belt"
  ],
  "previousOwners": [
    {
      "firstName": "Sam",
      "lastName": "Hamwich"
    },
    {
      "firstName": "Leigha",
      "lastName": "Tard"
    }
  ]
}})

Detailed information about an account. Each element in the "accts" array corresponds to a single account in the returned information. The "contact" element holds information about the primary account contact.

  • enabled - Is this account enabled?
  • autoThreshold - Auto-confirmation threshold for this account.
  • accountID - Account ID.
  • contact.company - Company.
  • contact.title - Title.

  • contact.middleName - Middle name.
  • contact.lastName - Last name.
  • contact.address1 - Address, first line.
  • contact.address2 - Address, second line.
  • contact.city - City.
  • contact.state - State.
  • contact.zip - ZIP code.
  • contact.eMail - Email address.
  • contact.homePhone - Home phone number.
  • contact.workPhone - Work phone number.
  • contact.faxNumber - Fax number.
  • contact.timezoneID - Time zone ID string (possible values are listed here).
  • userIDs[] - Array of user IDs of all users that are part of this account.
  • stationIDs[] - Array of FaceFirst tracking stream IDs attached to this account.
  • numCases - The number of FaceFirst cases owned by this account.
  • caseIDs[] - Array of FaceFirst case IDs of all cases owned by this account. Note: This will only be returned if specifically requested; see the queryAccount function for more details.
  • cgroupIDs[] - Array of case group IDs of all case groups defined by this account.
  • subIDs[] - Array of subscription IDs representing all of this account's subscriptions.
  • agroupIDs[] - Array of alert group IDs of all alert groups defined by this account.
  • maxTransactionAge - Maximum amount of time in the past to look for pending investigations (seconds).
  • maxInvestigationLength - Maximum amount of time between consecutive pieces of evidence for a single investigation (seconds).
  • maxFaceNoteTimeDiff - Maximum amount of time between the addition of a face to a case and the creation of a note associated with the same case (seconds).
  • sessionExpiration - Session idle timeout (seconds).
  • maxEnrollments - Maximum number of cases that can be enrolled for this account.
  • maxSubusers - Maximum number of users that can exist on this account.
  • authType - The type of authentication used for this account; 0 for FaceFirst authentication, 1 for LDAP authentication, 2 for TOTP authentication.
  • authParams - Authentication parameters. Undefined for FaceFirst authentication; contains the server configuration for LDAP authentication; can contain optional TOTP parameters, e.g. issuer or period
  • caseTTL - Default amount of time that a case should remain enabled after creation (seconds). If set to 0, the case will never be disabled; if < 0, the case will be created disabled.

EnrollmentLogEntries

  • enrolllog
    • page
      • subsetStart [ integer ]
      • subsetCount [ integer ]
      • totalCount [ integer ]
    • entries[]
      • entryID [ string ]

      • userID [ string ]

      • time [ integer ]

      • submittedImages [ integer ]
      • caseImages [ integer ]
      • newImages [ integer ]

A list of enrollment log entries. Each element in the "entries" array is a single log entry. Note that not all fields will be present in all entries, specifically, the submittedImages, caseImages and newImages fields. For 'create' and 'remove' events, the submittedImages and caseImages fields will be present. For 'add' events, the submittedImages field will be present as well as one of caseImages or newImages. If the biometric provider isn't cropping images (the default configuration), newImages will be present. Otherwise, caseImages will be present. None of submittedImages, newImages and caseImages will be present for a 'delete' event. The 'page' element contains paging subset info.

  • page.subsetStart - Offset into full result set of this subset (first is 0).
  • page.subsetCount - Number of entries in this subset.
  • page.totalCount - Number of entries in full result set.
  • entries[].entryID - Log entry ID.
  • entries[].accountID - Account ID.
  • entries[].userID - ID of user that performed the modification.
  • entries[].caseID - FaceFirst case ID that ws modified.
  • entries[].time - Time when the event occurred (milliseconds since epoch).
  • entries[].event - description of enrollment event. This will be one of 'create' (case enrolled), 'add' (images added to case), 'remove' (images removed from case), 'delete' (case deleted).
  • entries[].submittedImages - Number of images that were submitted for the event.
  • entries[].caseImages - Total number of images enrolled for the case after this event.
  • entries[].newImages - Number of new images added to the case after an 'add' event.

The actual metrics returned are provider dependent.

  • faces[].yaw - Yaw of the face (provider dependent).
  • faces[].pitch - Pitch of the face (provider dependent).
  • faces[].metrics[].name - The name of the metric.
  • faces[].metrics[].score - The raw score for the metric.
  • faces[].metrics[].pass - If true, the metric passed the minimum requirement for acceptability.

This holds a set of profiles. A profile is used as a template for creating case and note objects. The availability parameter indicates which object the profile is appropriate for. The actual profile is defined by a list of groups, each of which contains a list of fields. This allows the creation of a simple hierarchy of information to be stored for each object. In addition, the list_description is an array of values to be used for the profile in a list context, e.g., if a user interface displays a list of search results, the list_description could include a "Name" element specified as a combination of first and last name fields, and an element for the birth date of the person described by the case. For the "Name" element, the display field would be "Name", the format would be "%s %s", and the parameters would include the group/field of the first name followed by the group/field of the last name. Any number of entries can be defined for the list_description, however, if used in a list context, there is not likely to be space to display more than 2 or 3 values.
